Описание проекта
«Сложно сосредоточиться» — это адаптивный лендинг, демонстрирующий современные подходы к вёрстке с приоритетом доступности и пользовательского комфорта. Проект представляет собой информационный сайт о методах концентрации внимания в эпоху информационного шума.



Ключевые особенности:

Полностью адаптивная вёрстка с подходом Desktop First (3 ключевых разрешения: мобильные устройства, планшеты, десктопы)

Поддержка светлой и тёмной темы с автоматическим переключением через prefers-color-scheme

Ручное переключение темы через кнопки «День», «Неон», «Авто» с сохранением выбранного состояния

Семантическая вёрстка с учётом стандартов доступности (a11y)

Ленивая загрузка изображений галереи (loading="lazy")

Декоративные элементы (уголки, плашка REC), реализованные через CSS-псевдоэлементы без использования дополнительных изображений



Стек технологий:

HTML5 — семантическая разметка

CSS3 — кастомные свойства (CSS-переменные), гриды, флексы, медиазапросы

Адаптивная типографика — функция clamp() для резиновых шрифтов

Методология БЭМ — для именования классов (naming convention)

JavaScript — переключение тем (предоставлен в стартовом шаблоне)

Выводы по результатам проекта
В ходе выполнения проектной работы были достигнуты следующие результаты:

Реализована полноценная поддержка двух цветовых тем через систему CSS-переменных. Переопределение всего 8 переменных в файлах dark.css и light.css позволяет управлять цветовой схемой всего сайта, что делает процесс ребрендинга или добавления новых тем максимально простым.

Обеспечена бесшовная адаптивность для всех типов устройств. Благодаря подходу Mobile First и грамотно настроенным медиазапросам, сайт корректно отображается на экранах от 320px до 1920px и выше, без горизонтального скролла и «разрывов» вёрстки.

Повышена доступность сайта для пользователей с ограниченными возможностями. Использование семантических тегов, атрибутов alt для всех изображений, скрытие декоративного контента от скринридеров (aria-hidden) и правильная стилизация фокуса (:focus-visible) позволяют пользователям с клавиатурной навигацией и программами чтения с экрана полноценно взаимодействовать с сайтом.

Достигнута высокая производительность. Ленивая загрузка изображений в галерее сокращает начальный вес страницы и ускоряет отображение первого экрана.

Создана система переключения тем с визуальной обратной связью. Кнопки выбора темы имеют все необходимые состояния (активная, hover, focus), что делает интерфейс интуитивно понятным и «живым».

Реализованы сложные декоративные элементы (уголки в шапке и подвале, плашка REC с точкой) без использования лишних HTML-элементов и изображений — только CSS-псевдоэлементы и абсолютное позиционирование.



Инструкция по развёртыванию и системные требования
Системные требования
Современный веб-браузер с поддержкой:

CSS Custom Properties (переменные)

CSS Grid Layout

prefers-color-scheme медиафича

:focus-visible псевдокласса

loading="lazy" атрибут

Рекомендованные браузеры: Google Chrome (версия 89+), Mozilla Firefox (версия 88+), Safari (версия 15+), Edge (версия 90+)

Разрешение экрана: от 320px (минимальное поддерживаемое)

Среда разработки: VS Code с расширением Live Server (рекомендуется)



Инструкция по запуску
Клонировать репозиторий

bash
git clone <url-репозитория>
Перейти в директорию проекта

bash
cd slozhno-sosredotochitsya
Открыть проект в VS Code

bash
code .
Запустить Live Server

Нажать правой кнопкой мыши на файл index.html

Выбрать "Open with Live Server"

Или использовать сочетание клавиш: Alt + L затем Alt + O

После этого проект автоматически откроется в браузере по адресу: http://127.0.0.1:5500

Почему Live Server?
Live Server автоматически перезагружает страницу при изменениях в коде, что ускоряет разработку и тестирование. Кроме того, он корректно обрабатывает все относительные пути к файлам (стили, шрифты, изображения).
